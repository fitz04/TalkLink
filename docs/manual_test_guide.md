# TalkLink 수동 테스트 절차서

**테스트 대상**: 실시간 번역 채팅 (v1.0)
**시나리오**: 다국어(한국어 ↔ 영어) 사용자 간의 채팅 검증

## 사전 준비

1. **개발 서버 실행**:
    - 터미널에서 `npm run client` (포트 5173/5174) 실행 확인.
    - 터미널에서 `node server/index.js` (포트 3000) 실행 확인.
2. **브라우저 준비**:
    - 크롬 일반 모드 (사용자 A용)
    - 크롬 시크릿 모드 또는 다른 브라우저 (사용자 B용)

---

## 단계별 절차 (Step-by-Step)

### 1단계: 사용자 A (한국어/호스트) 로그인

1. 일반 브라우저에서 `http://localhost:5173` 접속.
2. **로그인** (또는 회원가입):
    - 이메일: `user_a_kr@talklink.com` (임의)
    - 비밀번호: `Password123!`
    - 이름: `User A (KR)`
3. **채팅방 입장**:
    - 사이드바에서 "Global Chat" 클릭 또는 "새 채팅방 만들기"로 생성.
    - URL의 채팅방 ID 확인 (예: `/room/1`).

### 2단계: 사용자 B (영어/게스트) 접속

1. 시크릿 탭(Incognito) 열기.
2. `http://localhost:5173` 접속.
3. **로그인** (또는 회원가입):
    - 이메일: `user_b_en@talklink.com`
    - 비밀번호: `Password123!`
    - 이름: `User B (EN)`
4. **채팅방 입장**:
    - 사용자 A와 **동일한 채팅방** ("Global Chat") 입장.
    - *Tip: 게스트 모드라면 1단계 방의 '초대 코드'를 입력해 접속.*

### 3단계: 대화 및 번역 검증

#### Case 1: 한국어 → 영어 (API 키 있을 때)

1. **[사용자 A]** 입력: `"안녕하세요, 반가워요."` 전송.
2. **[사용자 B]** 화면 확인:
    - 원문: "안녕하세요, 반가워요." (작게 표시되거나 옵션)
    - **번역**: *"Hello, nice to meet you."* (강조 표시)
    - *말풍선 스타일이 구별되는지 확인.*

#### Case 2: 영어 → 한국어

1. **[사용자 B]** 입력: `"The weather is nice today."` 전송.
2. **[사용자 A]** 화면 확인:
    - **번역**: *"오늘 날씨가 좋네요."*

#### Case 3: API 오류 상황 (Resilience Test)

*의도적으로 API 키를 제거하거나 잘못된 키로 설정한 후 테스트*

1. **[사용자 A]** 입력: `"번역이 안되더라도 전송되어야 합니다."`
2. **[사용자 B]** 화면 확인:
    - 번역문 없이 원문 플레인 텍스트가 즉시 표시됨.
    - 채팅이 멈추거나 에러 팝업으로 차단되지 않아야 함.

## 종료 기준 (Success Criteria)

- [ ] 두 사용자 간 메시지가 **실시간(1초 이내)**으로 전달되는가?
- [ ] API 키가 유효할 때 **번역된 텍스트**가 상대방에게 보이는가?
- [ ] 번역 실패 시에도 **채팅 기능(메시지 전송)**이 정상 작동하는가?
